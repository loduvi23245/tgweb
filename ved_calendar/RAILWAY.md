# Инструкция по запуску на Railway

## Подготовка

1. **Создайте аккаунт на Railway**
   - Перейдите на https://railway.app
   - Зарегистрируйтесь через GitHub

2. **Подготовьте репозиторий**
   - Убедитесь, что ваш код загружен на GitHub
   - В репозитории должны быть файлы:
     - `Dockerfile`
     - `railway.json`
     - `requirements.txt`
     - Все исходные файлы проекта

## Шаг 1: Создание проекта на Railway

### Вариант A: Один сервис (WebApp + Admin)

1. Нажмите **"New Project"**
2. Выберите **"Deploy from GitHub repo"**
3. Выберите ваш репозиторий с проектом
4. Railway автоматически обнаружит Dockerfile и начнет сборку

### Вариант B: Раздельные сервисы (рекомендуется для продакшена)

Создайте три отдельных сервиса в одном проекте Railway:

1. **WebApp Service** (веб-приложение)
   - Deploy Command: `python database/init_db.py && python webapp/main.py`
   - Порт: 8000

2. **Admin Service** (админ-панель)
   - Deploy Command: `python admin/main.py`
   - Порт: 8001

3. **Bot Service** (Telegram бот)
   - Deploy Command: `python bot/main.py`
   - Порт: не требуется (бот работает через polling)

## Шаг 2: Настройка базы данных PostgreSQL

1. В вашем проекте нажмите **"+"** → **"New"** → **"Database"** → **"PostgreSQL"**
2. Дождитесь создания базы данных
3. Railway автоматически создаст переменную окружения `DATABASE_URL`
4. Подключите базу данных ко всем сервисам (WebApp, Admin, Bot)

## Шаг 3: Настройка переменных окружения

В разделе **Variables** для каждого сервиса добавьте следующие переменные:

### Обязательные переменные (для всех сервисов):
```bash
TELEGRAM_BOT_TOKEN=ваш_токен_бота_от_BotFather
DATABASE_URL=автоматически_создается_Railway
ADMIN_USERNAME=admin
ADMIN_PASSWORD=ваш_сложный_пароль
ADMIN_SECRET_KEY=ваш_секретный_ключ_для_jwt
JWT_SECRET_KEY=ваш_секретный_ключ_для_jwt
DEBUG=False
```

### Дополнительные переменные:

**Для WebApp Service:**
```bash
PORT=8000
RAILWAY_PUBLIC_URL=https://ваш-домен.up.railway.app
```

**Для Admin Service:**
```bash
ADMIN_PORT=8001
```

**Для Bot Service:**
```bash
TELEGRAM_ADMIN_IDS=ваш_telegram_id (например: 123456789)
```

## Шаг 4: Настройка портов

Railway автоматически назначит порты. Убедитесь, что:
- **WebApp Service**: порт 8000 открыт для публичного доступа
- **Admin Service**: порт 8001 открыт для публичного доступа
- **Bot Service**: порты не требуются (работает через polling)

## Шаг 5: Запуск

1. После настройки переменных Railway автоматически перезапустит каждый сервис
2. Откройте логи (**Logs**) для каждого сервиса и убедитесь, что:
   - База данных инициализирована успешно
   - Бот запущен и подключен к Telegram
   - Веб-приложение доступно
   - Админ-панель доступна

## Шаг 6: Настройка Telegram бота

1. Откройте @BotFather в Telegram
2. Создайте нового бота или выберите существующего
3. Получите токен бота и сохраните его в переменных окружения
4. Настройте Menu Button для Web App:
   ```
   /setmenubutton → выберите бота → введите URL: https://ваш-домен.up.railway.app
   ```
5. (Опционально) Настройте описание бота:
   ```
   /setdescription → выберите бота → введите описание
   ```

## Шаг 7: Проверка работы

1. **Проверка бота:**
   - Откройте вашего бота в Telegram
   - Нажмите `/start`
   - Проверьте работу меню и команд

2. **Проверка Web App:**
   - Нажмите кнопку меню в боте
   - Должно открыться веб-приложение с мероприятиями

3. **Проверка админ-панели:**
   - Перейдите по URL: `https://ваш-домен-admin.up.railway.app/admin/login`
   - Войдите под созданным логином/паролем
   - Проверьте создание и модерацию мероприятий

## Переменные окружения Railway

Railway автоматически предоставляет некоторые переменные:

- `RAILWAY_PUBLIC_URL` - публичный URL вашего приложения
- `RAILWAY_PRIVATE_URL` - внутренний URL для связи между сервисами
- `DATABASE_URL` - URL подключения к PostgreSQL
- `PORT` - основной порт (по умолчанию 8000)

## Мониторинг и логи

- **Logs** - просмотр логов в реальном времени для каждого сервиса
- **Metrics** - метрики использования ресурсов (CPU, память)
- **Deployments** - история деплоев с возможностью отката

## Обновление проекта

При каждом пуше в ветку `main` (или другую настроенную):
1. Railway автоматически запустит новый деплой для каждого сервиса
2. Старая версия будет заменена новой
3. Если деплой неудачный, можно откатиться к предыдущей версии в разделе Deployments

## Решение проблем

### Ошибка подключения к базе данных
- Убедитесь, что создана база данных PostgreSQL
- Проверьте переменную `DATABASE_URL` во всех сервисах
- Убедитесь, что база данных подключена ко всем сервисам

### Бот не отвечает
- Проверьте `TELEGRAM_BOT_TOKEN`
- Убедитесь, что токен действителен
- Проверьте логи Bot Service на наличие ошибок
- Убедитесь, что у бота есть доступ к базе данных

### Не открывается Web App
- Проверьте `RAILWAY_PUBLIC_URL`
- Убедитесь, что порт 8000 открыт в настройках WebApp Service
- Проверьте CORS настройки в коде

### Доступ к админ-панели
- URL должен быть: `https://ваш-домен.up.railway.app/admin/login`
- Если используете отдельный сервис для админки: `https://ваш-домен-admin.up.railway.app/admin/login`
- Проверьте переменные `ADMIN_USERNAME` и `ADMIN_PASSWORD`

### Ошибки при деплое
- Проверьте логи сборки (Build Logs)
- Убедитесь, что все зависимости указаны в `requirements.txt`
- Проверьте синтаксис Python файлов

## Безопасность

1. **Используйте сложные пароли** для админ-панели
2. **Не коммитьте `.env` файл** с секретами в репозиторий
3. **Регулярно обновляйте зависимости** (`pip install --upgrade`)
4. **HTTPS включен автоматически** на Railway
5. **Используйте Secrets** в Railway для хранения чувствительных данных
6. **Ограничьте доступ** к админ-панели по IP (если возможно)

## Оптимизация производительности

1. **Кэширование**: включите кэширование событий на 1 час
2. **Пагинация**: используйте пагинацию для списков событий (20-50 записей)
3. **Индексы БД**: убедитесь, что в базе данных есть индексы для частых запросов
4. **Статические файлы**: используйте CDN для статических файлов (опционально)

## Резервное копирование

Railway автоматически создает резервные копии базы данных PostgreSQL. Для дополнительной надежности:

1. Экспортируйте данные регулярно через админ-панель
2. Используйте внешние сервисы для бэкапа БД
3. Сохраняйте конфигурацию переменных окружения

## Поддержка

При возникновении проблем:
1. Проверьте логи в Railway (**Logs**)
2. Убедитесь, что все переменные окружения установлены
3. Проверьте документацию Railway: https://docs.railway.app
4. Обратитесь в поддержку Railway через Discord
